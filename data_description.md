# LineVD 数据说明文档

## 1. 图对象 `g` 的属性

### 1.1 节点特征 (`g.ndata`)

| 特征名称 | 类型 | 维度 | 说明 |
|---------|------|------|------|
| `_LINE` | 张量 | [N] | 代码行号 |
| `_VULN` | 张量 | [N] | 漏洞标签（0=安全，1=漏洞） |
| `_FVULN` | 张量 | [N] | 函数级漏洞标签 |
| `_CODEBERT` | 张量 | [N, 768] | CodeBERT 嵌入特征 |
| `_GRAPHCODEBERT` | 张量 | [N, 768] | GraphCodeBERT 嵌入特征 |
| `_GLOVE` | 张量 | [N, 200] | GloVe 词向量特征 |
| `_DOC2VEC` | 张量 | [N, 300] | Doc2Vec 文档向量特征 |
| `_FUNC_EMB` | 张量 | [N, 768] | 函数级嵌入特征 |
| `_SASTRATS` | 张量 | [N] | RATS 工具的标签 |
| `_SASTCPPCHECK` | 张量 | [N] | cppcheck 工具的标签 |
| `_SASTFLAWFINDER` | 张量 | [N] | flawfinder 工具的标签 |
| `_RANDFEAT` | 张量 | [N, 200] | 随机特征 |

### 1.2 边特征 (`g.edata`)

| 特征名称 | 类型 | 维度 | 说明 |
|---------|------|------|------|
| `_TYPE` | 张量 | [E] | 边的类型（如依赖、控制流等） |

### 1.3 图结构属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `number_of_nodes()` | 方法 | 返回图中节点数量 |
| `number_of_edges()` | 方法 | 返回图中边数量 |
| `ndata` | 字典 | 节点特征字典 |
| `edata` | 字典 | 边特征字典 |
| `srcnodes` | 张量 | 边的源节点 |
| `dstnodes` | 张量 | 边的目标节点 |

### 1.4 图对象索引含义

当使用 `load_graphs()` 函数加载图时，返回的是一个元组，其中包含图对象和其他信息。具体含义如下：

| 表达式 | 含义 | 说明 |
|--------|------|------|
| `g` | 完整的返回结果 | 包含图对象和其他信息的元组 |
| `g[0]` | 图对象列表 | 包含所有加载的图对象 |
| `g[0][0]` | 第一个图对象 | 实际使用的 DGL 图对象 |
| `g[1]` | 图的标签信息 | 包含图的额外标签信息 |

**示例**：
```python
# 加载图
loaded = load_graphs("path/to/graph.bin")
g_list = loaded[0]  # 图对象列表
g = g_list[0]       # 第一个图对象
labels = loaded[1]   # 标签信息
```

## 2. `bigvul()` 处理后的 DataFrame `df` 数据项

### 2.1 基本信息

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `id` | 整数 | 样本唯一标识符 |
| `vul` | 整数 | 漏洞标签（0=安全，1=漏洞） |
| `cve` | 字符串 | CVE 编号 |
| `project` | 字符串 | 项目名称 |
| `commit_id` | 字符串 | 提交 ID |
| `file` | 字符串 | 文件名 |
| `func` | 字符串 | 函数名 |

### 2.2 代码信息

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `code` | 字符串 | 代码内容 |
| `code_before` | 字符串 | 漏洞修复前的代码 |
| `code_after` | 字符串 | 漏洞修复后的代码 |
| `line_start` | 整数 | 函数起始行号 |
| `line_end` | 整数 | 函数结束行号 |

### 2.3 漏洞信息

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `removed` | 列表 | 被移除的漏洞行号 |
| `added` | 列表 | 新添加的安全行号 |
| `diff` | 字符串 | 代码差异 |
| `cwe` | 字符串 | CWE 漏洞类型 |
| `severity` | 字符串 | 漏洞严重程度 |

### 2.4 处理状态

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `processed` | 布尔值 | 是否已处理 |
| `joern_status` | 字符串 | Joern 处理状态 |
| `graph_path` | 字符串 | 图文件路径 |
| `embedding_path` | 字符串 | 嵌入文件路径 |

### 2.5 数据集划分信息

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `label` | 字符串 | 数据集划分标签（train/val/test） |

### 2.6 其他可能的列

| 数据项 | 类型 | 说明 |
|--------|------|------|
| `func_start` | 整数 | 函数在文件中的起始行 |
| `func_end` | 整数 | 函数在文件中的结束行 |
| `num_lines` | 整数 | 代码行数 |
| `language` | 字符串 | 编程语言 |
| `fix_commit` | 字符串 | 修复提交 ID |

### 2.7 数据加载流程

1. **加载最小化数据集**：从 `minimal_bigvul_{sample}.pq` 文件加载
2. **删除空值**：执行 `dropna()` 操作
3. **加载元数据**：从 `bigvul_metadata.csv` 加载项目信息
4. **添加划分标签**：
   - 默认划分：从 `bigvul_rand_splits.csv` 加载
   - 跨项目划分：根据项目名称进行划分
5. **返回处理后的数据集**

### 2.8 特殊处理

- **id=177860**：该样本在 `before/after` 字段中不应该包含注释
- **跨项目划分**：支持 Linux、Chrome、Android、qemu 作为测试集的跨项目评估

## 3. 数据流程

### 3.1 数据加载流程

1. **原始数据**：从 BigVul 数据集加载原始数据
2. **预处理**：过滤、清洗和标准化数据
3. **最小化处理**：生成 `minimal_bigvul_{sample}.pq` 文件
4. **图构建**：使用 Joern 生成代码属性图
5. **嵌入生成**：生成 CodeBERT/GraphCodeBERT 嵌入
6. **特征添加**：添加 SAST 工具标签等特征

### 3.2 数据使用流程

1. **数据集初始化**：调用 `bigvul()` 加载数据
2. **图加载**：通过 `item()` 方法加载图对象
3. **特征处理**：处理嵌入特征和其他特征
4. **模型训练**：使用处理后的数据训练模型
5. **模型评估**：评估模型性能
